Failure analysis also matters here. The greedy baseline exhibits severe
degeneration: repeated 3-gram rate is nearly one, and lexical diversity is near
zero. This is exactly why quality-control metrics are necessary in the study:
otherwise a bias shift could be confounded by a generation-quality shift. In
addition, the scoring pipeline initially had a heavy model-loading path and an
expensive quality-bootstrap stage; both were simplified so that the experiment
could complete on the available hardware.

\section{Theoretical Discussion}
The main design choice in this project is to isolate decoding as the
experimental variable. The generator checkpoint remains fixed at GPT-2 small,
the prompt bank is fixed, and sample counts and maximum generation length are
held constant across decoding settings. This matters because the study is not
asking whether a different model is less biased in general; it is asking whether
sampling policy alone changes the regard profile of generated text.

Masking demographic mentions with \texttt{XYZ} before regard scoring is also an
important design choice. Without masking, the classifier may respond partly to
the surface demographic token itself instead of the semantic portrayal in the
continuation. The masking step therefore makes the score closer to a measure of
how the completion describes the demographic rather than whether the demographic
word appears.

The model behavior observed here is consistent with the decoding literature.
Greedy decoding collapses to repetitive high-probability continuations. Sampling
widens the lexical support and improves diversity, but this alone does not force
group-level regard gaps to vanish. In these results, quality improves
substantially while the selected negative-regard gap trace remains positive
across all decoding families. That pattern supports the project's central claim
that decoding affects bias behavior, but does not reduce the problem to a simple
``more diverse = unbiased'' story.

\section{Next Stage and Planned Improvements}
The next stage should focus on refinement rather than re-implementing the core
pipeline:
\begin{itemize}
\item strengthen Milestone 3 presentation with more compact plots derived from the
      combined metric files,
\item run the planned later-stage ablations, especially masking sensitivity and
      anti-repetition comparisons,
\item increase the quality-metric bootstrap count for final polished figures if
      runtime budget allows,
\item add a cleaner narrative linking quality change and bias change per prompt
      type, not only in global aggregates.
\end{itemize}

For future submissions, we also plan to add a direct comparison between GPT-2
and \texttt{Qwen3.5:0.8b}. That comparison is not claimed as part of the current
completed Milestone 2--3 scope, but it is a natural next experiment for testing
whether the decoding effects observed here generalize beyond GPT-2 small.
